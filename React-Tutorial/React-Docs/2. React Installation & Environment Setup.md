# ⚛️ React Installation & Environment Setup

## Prerequisites (Before Installing React)

Before creating a React project, we must install the following tools:

### Install Node.js

React applications run on **Node.js runtime environment**.

* Download from: [https://nodejs.org](https://nodejs.org)
* Install **LTS version (Recommended for production)**

### Verify Installation

Open terminal:

```bash
node -v
npm -v
```

This confirms Node.js and npm are installed.


### Install Code Editor

Recommended editor: **VS Code**

Why?

* Lightweight
* Extensions support
* Integrated terminal
* React IntelliSense

---

### Install Browser

Recommended: **Google Chrome**

Used for:

* Running React app
* Debugging
* React Developer Tools extension

---

# React Project Setup using Vite

## What is Vite

**Vite is a modern frontend build tool used to create fast React applications with instant development server startup and optimized production builds.**

It is faster than older tools like **Create React App (CRA)** because it uses **native ES Modules and lightning-fast bundling (esbuild).**


> Vite is a next-generation frontend tooling system that provides faster development and build performance for modern web applications.

---

## Ways to Create React Project using Vite

There are mainly **3 common ways**.

---

## Method 1 — Using npm (Most Common)

### Step-by-Step

```bash
npm create vite@latest
```

Then:

```bash
Project name → my-react-app
Framework → React
Variant → JavaScript / TypeScript
```

Now move into project:

```bash
cd my-react-app
npm install
npm run dev
```

Development server runs like:

```text
http://localhost:5173
```



> Default Vite dev server port is **5173**.

---

## Method 2 — Direct Command with Project Name

```bash
npm create vite@latest my-app -- --template react
```

Then:

```bash
cd my-app
npm install
npm run dev
```

Faster way (used by experienced developers)

---

## Method 3 — Using Yarn

If using yarn package manager:

```bash
yarn create vite
```

OR

```bash
yarn create vite my-app --template react
```

Then:

```bash
cd my-app
yarn
yarn dev
```

---

## Method 4 — Using pnpm (Modern Teams Use)

```bash
pnpm create vite
```

OR

```bash
pnpm create vite my-app --template react
```

Then:

```bash
cd my-app
pnpm install
pnpm dev
```

---

#  Root Level Folder & Files 


## node_modules

### What it actually contains

* All installed npm packages
* Dependency dependency (nested packages)
* Build tool internal code

When you run:

```bash
npm install
```

npm reads **package.json → downloads packages → stores here**



> node_modules is a dependency tree automatically generated by npm and should never be manually modified.

Also never push fully to GitHub (use `.gitignore`).

---

## public Folder

### Purpose

Stores **static assets that should not be processed by bundler**

Examples:

* favicon
* static images
* robots.txt
* manifest.json



Files inside public are served **as-is** by dev server.

React bundler **does not optimize or hash them**.

Used when:

* File must keep original name
* SEO files
* external integrations

---

## src Folder (Application Core)

This is the **heart of React application**

Everything related to:

* UI
* Business logic
* state
* hooks
* routing

lives inside src.

### Interview Statement

> src folder contains the complete component tree and application logic which gets bundled into production build.

---

# main.jsx — Application Bootstrap File

### Core Responsibility

This file **connects React with Browser DOM**

It is responsible for:

* Creating React Root
* Rendering App component
* Enabling Strict Mode

Conceptual flow:

```jsx
ReactDOM.createRoot(rootElement).render(<App />);
```


This is called **Application Mounting Phase**

Browser → loads index.html
→ main.jsx executes
→ React renders App
→ UI appears

---

# App.jsx — Root Component Architecture

### Role

Acts as:

* **Component Tree Parent**
* **Layout Controller**
* **Route Container**

Real-world responsibilities:

* Page routing
* global layout
* context providers
* theme wrappers



> App component serves as the top-level component managing overall UI composition.

---

## assets Folder

Stores:

* images
* svg
* fonts
* media


Bundler **optimizes and hashes these files** in production.

---

## Global Styles (index.css / main.css)

### Purpose

* CSS Reset
* typography system
* design tokens
* root variables

Applied to entire app.

---

## index.html — SPA Container

### Role

This is **single HTML entry file**

Contains:

```html
<div id="root"></div>
```

React injects complete UI inside this.


React follows **Client Side Rendering SPA model**

Meaning:

* Page never reloads
* Only components update

---

## package.json — Project Brain

### Stores

* dependencies
* scripts
* project metadata
* build configuration

Example scripts:

```json
"scripts": {
  "dev": "vite",
  "build": "vite build"
}
```


> package.json defines dependency graph and executable lifecycle scripts of the application.

---

## vite.config.js — Build System Controller

### Used for

* plugin configuration
* path aliases
* proxy setup
* build optimization

Real-world example:

API proxy to avoid CORS

---

# Complete Internal Flow (Very Important)

```text
User opens browser
   ↓
index.html loads
   ↓
main.jsx executes
   ↓
React creates Virtual DOM
   ↓
App component renders
   ↓
Child components render
   ↓
UI visible on screen
```
