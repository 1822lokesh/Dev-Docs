# SQL Keys 

**Keys in SQL** are constraints used to **uniquely identify records, maintain relationships between tables, and enforce data integrity.**

Keys help to:
- Prevent duplicate data
- Establish relationships between tables
- Improve query performance
- Maintain data consistency

**Interview Tip:**  
A key is an attribute or set of attributes that helps to **uniquely identify a row in a table.**

---

## Types of SQL Keys

1️⃣ Primary Key  
2️⃣ Foreign Key  
3️⃣ Unique Key  
4️⃣ Candidate Key  
5️⃣ Alternate Key  
6️⃣ Composite Key  
7️⃣ Super Key

## Primary Key

A **Primary Key** is a column that **uniquely identifies each record in a table.**

Rules:
- Cannot contain NULL values  
- Cannot contain duplicate values  
- Only one primary key per table  

### Example

```sql
CREATE TABLE Student (
    RollNo INT PRIMARY KEY,
    Name VARCHAR(50),
    Email VARCHAR(100),
    City VARCHAR(50)
);
````

---

## Foreign Key

A **Foreign Key** is used to **create relationship between two tables.**

It references the **Primary Key of another table.**

### Example

```sql
CREATE TABLE Enrollment (
    RollNo INT,
    CourseID INT,
    FOREIGN KEY (RollNo) REFERENCES Student(RollNo)
);
```

Ensures that RollNo must exist in Student table.

---

## Unique Key

Ensures that **all values in a column are unique**.

Difference from Primary Key:

* Unique key can accept **one NULL value** (MySQL concept)

### Example

```sql
CREATE TABLE Student (
    RollNo INT PRIMARY KEY,
    Email VARCHAR(100) UNIQUE
);
```

---

## Candidate Key

A **Candidate Key** is a column that **can qualify as a Primary Key**.

Example:

* RollNo
* Email

Both can uniquely identify a student → so both are candidate keys.

---

## Alternate Key

Candidate keys **not selected as primary key** are called **Alternate Keys**.

Example:

* RollNo → Primary Key
* Email → Alternate Key

---

## Composite Key

A key formed using **two or more columns**.

### Example

```sql
CREATE TABLE Enrollment (
    RollNo INT,
    CourseID INT,
    PRIMARY KEY (RollNo, CourseID)
);
```

Combination uniquely identifies record.

---

## Super Key

A **Super Key** is any column or group of columns that **uniquely identifies a row.**

Example super keys:

* RollNo
* RollNo + Name
* RollNo + Email

---

## Summary

* Primary Key → unique + not null
* Foreign Key → relationship between tables
* Unique Key → unique values allowed
* Candidate Key → possible primary keys
* Alternate Key → unused candidate key
* Composite Key → multiple columns key
* Super Key → any unique identifier

Keys are used to **maintain data integrity and relationships**
