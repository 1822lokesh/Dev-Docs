# SQL Commands

### **SQL commands are instructions used to interact with a relational database management system, categorized into five main types: DDL, DQL, DML, DCL, and TCL.**

## DQL (Data Query Language)

**DQL (Data Query Language)** is a category of SQL commands used **to retrieve (fetch) data from the database without modifying it**.

**DQL mainly consists of only one command:**

```
SELECT
````

Other keywords such as `FROM`, `WHERE`, `GROUP BY`, `HAVING`, `ORDER BY`, `DISTINCT`, and `LIMIT` are **clauses used with SELECT**, not separate commands.

##  Clauses
* **SQL clauses are  fundamental components of SQL statements that define how data is retrieved, filtered, grouped, and organized from a database.** 
* They act as conditional statements, enabling users to apply constraints and perform specific operations on the data.

---

## Basic SELECT (Fetch All Data)

```sql
SELECT * FROM Student;
````

Returns **all columns and all rows** from the table.

---

## SELECT Specific Columns

```sql
SELECT Name, Marks FROM Student;
```

Returns only **Name and Marks columns**.

---

## WHERE Clause (Filtering Data)

```sql
SELECT * 
FROM Student
WHERE City = 'Bangalore';
```

Fetches students **only from Bangalore**.

---

## DISTINCT Clause (Unique Values)

```sql
SELECT DISTINCT City 
FROM Student;
```

Shows **unique cities** without duplicates.

---

## ORDER BY Clause (Sorting)

```sql
SELECT * 
FROM Student
ORDER BY Marks DESC;
```

Sorts students by **highest marks first**.

---

## GROUP BY Clause (Grouping Data)

```sql
SELECT City, COUNT(*) AS Total_Students
FROM Student
GROUP BY City;
```

Shows **number of students in each city**.

---

## HAVING Clause (Filter Groups)

```sql
SELECT City, AVG(Marks) AS Avg_Marks
FROM Student
GROUP BY City
HAVING AVG(Marks) > 70;
```

Shows cities where **average marks > 70**.

---

## LIMIT Clause (Restrict Rows)

```sql
SELECT * 
FROM Student
LIMIT 3;
```

Returns **first 3 rows only**.





## DDL (Data Definition Language)
**DDL (Data Definition Language)** is a category of SQL commands used **to define, create, modify, and delete the structure of database objects** such as tables, databases, indexes, etc.

DDL commands **affect the schema (structure) of the database**, not the actual data stored inside the tables.

**Interview Tip:**  
DDL is used to manage database structure. It includes commands like CREATE, ALTER, DROP, TRUNCATE, and RENAME.

---

## Common DDL Commands

- CREATE  
- ALTER  
- DROP  
- TRUNCATE  
- RENAME  

---

## CREATE Command (Create Database Object)

Used to **create a new table or database**

### Example: Create Student Table

```sql
CREATE TABLE Student (
    RollNo INT PRIMARY KEY,
    Name VARCHAR(50),
    Marks INT,
    Grade CHAR(1),
    City VARCHAR(50)
);
````

This command creates a **new table structure** in the database.

---

## ALTER Command (Modify Table Structure)

Used to **add, delete, or modify columns**

### Example: Add New Column

```sql
ALTER TABLE Student
ADD Age INT;
```
Adds a new column **Age** to the table.

---

## Example :Deleting Column

Table Name: Student

| RollNo | Name  | Marks | Grade | City | Age |
|--------|-------|-------|-------|------|-----|
| 1      | Ravi  | 85    | A     | BLR  | 20  |
| 2      | Anu   | 72    | B     | MYS  | 21  |
| 3      | Kiran | 90    | A     | BLR  | 22  |

---

## SQL Query to Delete Column

```sql
ALTER TABLE Student
DROP COLUMN Age;
```
Deletes the entire Age column

### Example: Modify Column

```sql
ALTER TABLE Student
MODIFY Name VARCHAR(100);
```

Changes column size.

---

## DROP Command (Delete Entire Object)

Used to **permanently delete a table or database**

```sql
DROP TABLE Student;
```

Deletes **table structure + all data permanently**.

⚠ Cannot be rolled back in most databases.

---

## TRUNCATE Command (Delete All Rows)

Used to **remove all records from a table but keep structure**

```sql
TRUNCATE TABLE Student;
```

Faster than DELETE
Resets auto-increment values
Structure remains same

---

## RENAME Command (Rename Object)

Used to **change table name**

```sql
RENAME TABLE Student TO Student_Details;
```

Only table name changes
Data and structure remain same




## DML (Data Manipulation Language)
**DML (Data Manipulation Language)** is a category of SQL commands used **to insert, update, and delete data stored inside database tables.**

Unlike DDL, DML **does not change the table structure**,  
it only **modifies the records (rows) present in the table.**

**Interview Tip:**  
DML is used to manage data inside tables. Common DML commands are INSERT, UPDATE, and DELETE.

---

## Common DML Commands

- INSERT  
- UPDATE  
- DELETE  

---

## Example Table

Table Name: Student

| RollNo | Name   | Marks | Grade | City      |
|--------|--------|-------|-------|-----------|
 


## INSERT Command (Add New Records)

Used to **insert new rows into a table**

```sql
INSERT INTO Student (RollNo, Name, Marks, Grade, City)
VALUES (1, 'Ravi', 85, 'A', 'Bangalore');
````

Adds a **new student record** to the table.

### Insert Multiple Rows

```sql id="dmlins2"
INSERT INTO Student 
VALUES 
(2, 'Anu', 72, 'B', 'Mysore'),
(3, 'Kiran', 90, 'A', 'Bangalore');
```

---

## UPDATE Command (Modify Existing Data)

Used to **change existing record values**

```sql id="dmlupd1"
UPDATE Student
SET Marks = 95
WHERE RollNo = 3;
```

Updates **marks of student with RollNo = 3**

⚠ Without WHERE → all rows will be updated (important interview point)

---

## DELETE Command (Remove Records)

Used to **delete specific rows from a table**

```sql id="dmldel1"
DELETE FROM Student
WHERE RollNo = 2;
```

Deletes **student record whose RollNo = 2**

### Delete All Rows

```sql id="dmldel2"
DELETE FROM Student;
```

Removes **all records but table structure remains**


## DCL (Data Control Language)
  **DCL (Data Control Language)** is a category of SQL commands used **to control access and permissions on database objects such as tables, views, procedures, etc.**

It helps the database administrator **define who can access the data and what operations they can perform.**
 
DCL is mainly used for **security and authorization management** in a database system.

---

## Common DCL Commands

- GRANT  
- REVOKE  

---

## GRANT Command (Give Permissions)

Used to **provide specific privileges to a user**

### Example: Give SELECT Permission

```sql
GRANT SELECT 
ON Student 
TO user1;
````

Now **user1 can read data** from the Student table.

### Example: Give Multiple Permissions

```sql id="dclgr2"
GRANT INSERT, UPDATE 
ON Student 
TO user1;
```

user1 can now **insert and update records**.

---

## REVOKE Command (Remove Permissions)

Used to **remove previously granted privileges**

```sql id="dclrv1"
REVOKE INSERT, UPDATE 
ON Student 
FROM user1;
```

user1 can **no longer insert or update data**.

---

## DCL Key Points 

* Used for **database security and access control**
* Controls **user privileges**
* Mainly handled by **Database Administrator (DBA)**
* Important commands →
  `GRANT`, `REVOKE`



## TCL (Transaction Control Language)

**TCL (Transaction Control Language)** is a category of SQL commands used **to manage transactions in a database.**

A **transaction** is a group of one or more SQL operations executed as a single unit of work.

TCL ensures **data consistency and integrity** by allowing changes to be **saved permanently or undone.**

 
TCL is used to control DML operations using commands like COMMIT, ROLLBACK, SAVEPOINT, and START TRANSACTION.

---

## Common TCL Commands

- START TRANSACTION / BEGIN  
- COMMIT  
- ROLLBACK  
- SAVEPOINT  

---

## Example Table

Table Name: Student

| RollNo | Name  | Marks | Grade | City |
|--------|-------|-------|-------|------|



## START TRANSACTION

Used to **begin a transaction**

```sql
START TRANSACTION;
````

After this, SQL statements become part of a transaction.

---

## COMMIT Command (Save Changes Permanently)

```sql
INSERT INTO Student
VALUES (1, 'Ravi', 85, 'A', 'Bangalore');

COMMIT;
```

* Changes are **saved permanently in database**
* Cannot be rolled back after commit

---

## ROLLBACK Command (Undo Changes)

```sql id="tclrb1"
INSERT INTO Student
VALUES (2, 'Anu', 72, 'B', 'Mysore');

ROLLBACK;
```

Insert operation is **cancelled / undone**

---

## SAVEPOINT Command (Create Checkpoint)

```sql id="tclsv1"
START TRANSACTION;

INSERT INTO Student VALUES (3, 'Kiran', 90, 'A', 'Bangalore');

SAVEPOINT sp1;

UPDATE Student
SET Marks = 95
WHERE RollNo = 3;

ROLLBACK TO sp1;

COMMIT;
```

* Rollback happens **only up to savepoint**
* Insert remains
* Update is undone

---

## TCL Key Points (Interview Revision)

* Used to **manage transactions**
* Helps maintain **data integrity**
* Works mainly with **DML operations**
* Important commands →
  `COMMIT`, `ROLLBACK`, `SAVEPOINT`, `START TRANSACTION`

---

## Difference Between DROP, DELETE, and TRUNCATE in SQL

These commands are used to **remove data or database objects**, but they work in **different ways**.

---

## 1. DELETE

- Used to **remove specific rows** from a table  
- Can use **WHERE condition**  
- Table structure **remains same**  
- Can be **rolled back (in transaction)**  
- Slower compared to TRUNCATE  

### Example

```sql
DELETE FROM Student
WHERE id = 1;
````

Deletes only the row where **id = 1**

---

## 2. TRUNCATE

* Used to **remove all rows** from a table
* **WHERE condition cannot be used**
* Table structure **remains same**
* Faster than DELETE
* Cannot be rolled back (generally)

### Example

```sql
TRUNCATE TABLE Student;
```

Deletes **all records** from Student table

---

## 3. DROP

* Used to **delete entire table or database**
* Removes **table structure + data**
* Cannot be rolled back
* Frees memory completely

### Example

```sql
DROP TABLE Student;
```

Deletes **complete table permanently**

---


## DROP vs DELETE vs TRUNCATE (Comparison)

| Feature | DELETE | TRUNCATE | DROP |
|--------|--------|----------|------|
| Command Type | DML (Data Manipulation Language) | DDL (Data Definition Language) | DDL (Data Definition Language) |
| Purpose | Remove specific rows from table | Remove all rows from table | Remove entire table/database |
| WHERE Clause | Supported | Not Supported | Not Supported |
| Removes All Rows | Yes (without WHERE) | Yes | Yes |
| Removes Selected Rows | Yes | No | No |
| Table Structure | Remains | Remains | Deleted |
| Memory Space | Not fully freed | Freed |  Fully freed |
| Speed | Slow (row by row) | Fast | Very Fast |
| Rollback Possible |  Yes (if transaction used) |  No (generally) |  No |
| Trigger Activation |Yes |  No | No |
| Auto Increment Reset | No | Yes | Yes (table removed) |
| Usage Risk Level | Low | Medium | High |
