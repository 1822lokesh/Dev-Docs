
## SQL Constraints

SQL Constraints are rules applied on table columns to **control the type of data**  
that can be inserted into a table.

They help to **maintain data accuracy, consistency, and integrity**.

---

## Types of SQL Constraints

- NOT NULL  
- UNIQUE  
- PRIMARY KEY  
- FOREIGN KEY  
- CHECK  
- DEFAULT  

---

## 1. NOT NULL Constraint

Ensures that a column **cannot store NULL (empty) values**.

### Example

```sql
CREATE TABLE Student (
    id INT,
    name VARCHAR(50) NOT NULL,
    marks INT
);
````

Here **name must have a value**.

---

## 2. UNIQUE Constraint

Ensures that all values in a column are **different (no duplicates)**.

### Example

```sql
CREATE TABLE Student (
    id INT,
    email VARCHAR(50) UNIQUE
);
```

Email must be **unique for each record**.

---

## 3. PRIMARY KEY Constraint

* Uniquely identifies each row
* Cannot contain NULL
* Only **one primary key** per table

### Example

```sql
CREATE TABLE Student (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);
```

---

## 4. FOREIGN KEY Constraint

Used to **link two tables**.
Maintains **referential integrity**.

### Example

```sql
CREATE TABLE Orders (
    order_id INT PRIMARY KEY,
    student_id INT,
    FOREIGN KEY (student_id) REFERENCES Student(id)
);
```

 student_id must exist in **Student table**.

---

## 5. CHECK Constraint

Ensures that values satisfy a **specific condition**.

### Example

```sql
CREATE TABLE Student (
    id INT,
    marks INT CHECK (marks >= 35)
);
```

 Marks must be **greater than or equal to 35**.

---

## 6. DEFAULT Constraint

Assigns a **default value** if no value is provided.

### Example

```sql
CREATE TABLE Student (
    id INT,
    city VARCHAR(50) DEFAULT 'Bangalore'
);
```

If city not given → automatically **Bangalore**.
