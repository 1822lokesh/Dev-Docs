# SQL Data Types

SQL Data Types define the **type of data** that can be stored in a table column.

They help the database:

- Store data efficiently  
- Maintain data accuracy  
- Improve performance  

---

## 1. Numeric Data Types

Used to store **numbers**.

| Data Type | Description | Example |
|----------|------------|---------|
| INT | Stores whole numbers | 10, 200, -5 |
| BIGINT | Stores very large integers | 999999999 |
| FLOAT | Stores decimal numbers (approximate) | 10.5, 23.78 |
| DOUBLE | Stores large decimal values | 3456.789 |
| DECIMAL(p,s) | Exact decimal values (precision & scale) | 99.99 |

---

## 2. String (Character) Data Types

Used to store **text values**.

| Data Type | Description | Example |
|----------|------------|---------|
| CHAR(n) | Fixed length string | 'A', 'INDIA' |
| VARCHAR(n) | Variable length string | 'Arun' |
| TEXT | Stores large text data | Paragraph |

---

## 3. Date and Time Data Types

Used to store **date and time values**.

| Data Type | Description | Example |
|----------|------------|---------|
| DATE | Stores date (YYYY-MM-DD) | 2026-03-13 |
| TIME | Stores time (HH:MM:SS) | 10:30:00 |
| DATETIME | Stores date and time | 2026-03-13 10:30:00 |
| TIMESTAMP | Stores date & time (auto update) | Current system time |
| YEAR | Stores year value | 2026 |

---

## 4. Boolean Data Type

Used to store **true or false values**.

| Data Type | Description | Example |
|----------|------------|---------|
| BOOLEAN / BOOL | Stores TRUE (1) or FALSE (0) | TRUE |

---
